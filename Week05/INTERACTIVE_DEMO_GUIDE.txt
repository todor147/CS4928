OPTION 1: CREATE CUSTOM ORDER (INTERACTIVE)
================================================================================

This option guides you through building an order step-by-step:

STEP 1: Choose Your Base Drink
  • Espresso ($2.50)
  • Latte ($3.20)
  • Cappuccino ($3.00)

STEP 2: Add Optional Features
  • Extra Shot? (+$0.80)
  • Oat Milk? (+$0.50)
  • Syrup? (+$0.40)
  • Large Size? (+$0.70)

STEP 3: Set Quantity
  • Enter the number of this item

STEP 4: Add More Items?
  • Choose to add another item or proceed to checkout

STEP 5: Payment (Optional)
  • Cash
  • Card (enter card number)
  • Wallet (enter wallet ID)

Example Flow:
  1. Select option 1
  2. Choose Espresso (option 1)
  3. Add Extra Shot (y)
  4. Add Oat Milk (y)
  5. Skip Syrup (n)
  6. Skip Large (n)
  7. Quantity: 2
  8. Don't add more items (n)
  9. View order summary
  10. Process payment (optional)

Result: 2 × Espresso + Extra Shot + Oat Milk ($7.60 + tax)

  LAT - Latte

Add-on Codes:

PAYMENT PROCESSING
================================================================================

After viewing an order summary, you can optionally process payment:

Payment Methods:
  1. Cash - Simple cash payment
  2. Card - Requires card number (last 4 digits shown masked)
  3. Wallet - Requires wallet ID

Example:

  Enter card number: 1234567890123456

  ✓ Payment processed successfully!

================================================================================
DESIGN PATTERNS DEMONSTRATED
================================================================================

While using the interactive demo, you're experiencing:

1. DECORATOR PATTERN
   Each add-on (SHOT, OAT, SYP, L) is a decorator that wraps the base product
   and adds its functionality and price dynamically.

2. FACTORY PATTERN
   The ProductFactory creates products from recipe codes, hiding the
   complexity of constructing decorated products.

3. STRATEGY PATTERN
   Payment methods (Cash, Card, Wallet) implement different strategies
   for processing payments.

4. OBSERVER PATTERN
   When items are added to orders or payments are processed, observers
   are notified (though not visible in this CLI demo).

================================================================================
TROUBLESHOOTING
================================================================================

Q: Recipe code not working?
A: Make sure you're using valid codes (ESP, LAT, CAP for base; SHOT, OAT,
   SYP, L for add-ons). Check Option 4 for reference.

Q: Numbers not being accepted?
A: Enter just the number without any extra characters. Invalid entries
   default to safe values (usually 1).

Q: Want to start over?
A: Just select option 5 to exit and restart the demo, or complete your
   current order and start a new one from the main menu.

Q: Can I skip payment?
A: Yes! When asked "Process payment? [y/N]:", just press Enter or type 'n'.
